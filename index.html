由校： 实现一个体验监控系统

阿里数字供应链事业部

体验监控系统包括哪些方面

  • “用户体验”包含用户与公司的产品与服务交互中的所有面向，在任务中的学习性(Learnability)、性能(effectiveness)和效率(efficiency)、记忆性(Memorability)及记忆延续性(retainability)、错误率(error rate)及错误承受能力(error tolerance)
  • APM ( Application Performance Management) :对应用系统的性能和故障进行系统化的管理
  • 前端领域APM包括稳定性和流畅性两部分

体验监控系统怎么做

  • 系统架构

  • 日志的采集（稳定性）：
    • 脚本异常:页面内发生的JS报错
      • 全局监听error、unhandledrejection事件
      • window . addEventListener( 'error',()=>{}) window. addEventListener( ' unhandledrejection' ,()=>{})
    • 接口异常：监控页面内发出的接口请求的耗时和异常
      • 通过重写XMLHttpRequest和fetch的原生方法来实现
    • 资源异常：页面内的图片、CSS、 JS等Assets资源加载失败
      • 在捕获类型的error事件里可以拿到资源加载失败回调
    • 白屏：页面加载过程中因为任意问题导致渲染没有完成，呈现空白页面的异常
    • Crash：页面因为内存溢出、死循环等原因导致崩溃的异常
  • 日志采集（流畅性）：
    • 加载性能：
    • 操作流畅性：
      • 响应速度：用户操作到页面响应<100ms
      • 滚动流畅性：用户手指滑动到页面滚动<100ms
      • 动画流畅性：动画出现掉帧
      • 卡顿：UI线程存在50ms以上阻塞。（有浏览器的事件队列机制决定
  • 上报方案：
  • 日志的解析处理存储 - 分析、告警
    • 要做的事
      • 日志接收处理：建设后端应用，提供日志上报接口给到采集SDK
      • 数据发布：后端应用接收到的日志后处理成可被实时流计算处理的流式数据，如DataHub、SLS、Kafaka等
      • 日志处理：流计算平台对流式数据进行实时处理，可基于Flink、
        Spark、Storm等
        • 基于样本的实时分析：基于OLAP数据库进行实时分析，如阿里云Hologres、 Hive、 Kylin等
        • 实时汇总计算：将原始日志实时汇总计算成指标(如错误率)后存入OLTP数据库，如阿里云RDS、GaussDB、TBase等
      • 监控告警：建设前台应用，实现监控、告警
    • 需要的系统服务：
      • 应用服务器+ DataHub + Flink + Hologres ( RDS )
    • 云服务：基于阿里云提供的日志服务(SLS)快速实现日志的上报、分析和告警

自羽：微前端

飞猪业务技术部

微前端价值

  • 服务端的“微服务架构”：为了解决庞大的一-整块后端服务带来的变更与扩展方面的限制，出现了微服务架构( Microservices)。
    • 然而越来越重的前端工程也面临同样的问题，自然地想到了将微服务思想应用(照搬)到前端，于是有了「微前端(micro-frontends)」的概念
  • 前端的“微前端”：即，一种由独立交付的多个前端应用组成整体的架构风格。具体的，将前端应用分解成一些更小、更简单的能够独立开发、测试、部署的小块，而在用户看来仍然是内聚的单个产品。
